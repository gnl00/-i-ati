# dao

`dao/` 只放纯 SQL 访问层，职责是表级 CRUD。

适合放在这里的内容：
- `prepare` / `get` / `all` / `run` 这类直接面向表结构的查询与写入
- `Row` 级别的数据读写
- 与单表或少量紧邻表强相关、但仍然是 SQL 形状的问题

不应该放在这里的内容：
- 领域实体拼装
- 跨多个 dao 的业务组合
- 默认值策略、时间源注入策略、业务级校验
- “顺手”更新别的聚合状态

约束：
- dao 返回的应是 row / 原始持久化形状，而不是高层领域实体
- dao 可以知道表结构，但不应该知道 host、tool、runtime 之类上层语义
- 如果某段逻辑需要协调多个表，优先上移到 `repositories/`

判断标准：
- 如果删除 SQL 之后这个模块就不成立，它大概率属于 `dao/`
- 如果开始出现“为了业务方便，顺便再做一步”的冲动，这段逻辑通常已经不该留在 `dao/`
