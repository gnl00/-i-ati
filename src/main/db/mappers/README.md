# mappers

`mappers/` 放持久化形状与领域形状之间的显式转换。

适合放在这里的内容：
- `Row -> Entity`
- `Entity/Input -> Row`
- JSON 字段、枚举字段、压缩字段、协议字段的编解码

不应该放在这里的内容：
- SQL 查询
- 事务
- 跨 repository/dao 的编排
- 依赖外部服务的业务逻辑

约束：
- mapper 应尽量保持纯函数化，输入输出明确，可单测
- mapper 负责形状转换，不负责“何时读写数据库”
- 如果一个转换必须访问数据库或依赖运行时上下文，它通常不该是 mapper

判断标准：
- 如果一段逻辑本质是在回答“数据库里怎么存 / 代码里怎么用”，它适合放在 `mappers/`
- 如果一段逻辑开始决定“什么时候更新、更新几张表、失败怎么回滚”，它已经越过 mapper 边界了
